UNIT – 1: HTML BASICS & STRUCTURE

1. What is HTML? (HyperText Markup Language)

−

HTML is the standard language used to create the structure of webpages. It defines what appears on the browser-headings, text, images, videos, tables, buttons, and forms. HTML does not control styling (CSS does) or dynamic behavior (JS does). When a webpage loads, the browser reads the HTML first and converts it into a DOM (Document Object Model), a tree-like structure where every tag becomes a node.

Examples :

<h1>Hello World</h1>
<p>This is my first webpage.</p>

2. How Browser, Server & HTML Work Together

+

When you type a website URL: 

    User  ----  browser  ----  Server  ----  Html/CSS/JS files  ---  Browser Renders Page

- The browser sends the requist to the Server
- The server sends the html (Structure) / CSS (Styling) / JS (behavior).
- The browser constructs the DOM.
- CSS styling is applied.
- Java script adds interactivity. (click, hover, animations).

6. Lists (Ordered, Unordered, Nested)

+

Lists are used to group related items like steps, menus, features, or topics. Each item is wrapped inside an <li> (list item) tag.

Unordered List :

Displays bullet points (●) without any specific order.

<ul>
    <li>HTML</li>
    <li>CSS</li>
</ul>

Ordered List :

Displays items in a numbered order (1, 2, 3...). Useful for steps or rankings.

<ol>
    <li>Step 1</li>
    <li>Step 2</li>
</ol>

Nested List :

A list inside another list. Helpful to show sub-topics or categories.

<ul>
    <li>Frontend
        <ul>
            <li>HTML</li>
            <li>CSS</li>
        </ul>
    </li>
</ul>

7. HTML Attributes

+

HTML attributes are extra pieces of information added to tags to control their behavior, appearance, or identity. They are always written inside the opening tag in name="value" format.

Attributes help identify elements, link files, add styling hooks, and create tooltips or labels.

Example :

<img src="image.jpg" alt="My Photo" width="200">
<p id="msg" class="highlight" title="Greeting">Hello</p>

Common Attributes :

id – unique identifier for a single element
class – groups elements for CSS styling
src – source path for images, audio, video, etc.
href – URL for links in the <a> tag
alt – alternative text for images (important for accessibility)
title – shows tooltip text on hover
type – defines input type in forms (text, password, email, etc.)

8. Semantic Tags

+

Semantic tags give meaningful names to parts of a webpage. Unlike a generic <div> , they tell the browser and search engines what type of content is inside. This improves SEO, accessibility, and code readability.

Common semantic tags include <header> , <footer> , <main> , <section> , <nav> , <article> , and <aside> .

For example, <article> represents a self‑contained piece of content like a blog post, while <section> groups related content.

Examples :

<header>Website Header</header>
<article>News Article Content</article>

9. Structure of an HTML Document

+

A complete HTML page follows a fixed structure so the browser knows how to interpret it. It starts with the <!DOCTYPE html> declaration which tells the browser this is an HTML5 document.

The root <html> element contains two main sections: <head> and <body> . The <head> stores metadata like the page title, CSS links, scripts, and SEO tags, while the <body> holds everything the user sees—text, images, buttons, forms, and links.

Basic HTML Skeleton :

<!DOCTYPE html>
<html>
  <head>
    <title>My Page</title>
  </head>
  <body>
    <h1>Welcome!</h1>
  </body>
</html>

10. Purpose of <head> and <body>

+

The <head> section contains information about the webpage but is not visible to the user. It includes metadata, page title, favicon, SEO keywords, external CSS/JS links, and character set definitions.

The <body> section contains all visible content such as headings, paragraphs, images, buttons, forms, and any scripts that run on the page. Browsers read the <head> first to prepare the page and then display the <body> content.

Both sections are essential for organizing and loading a webpage correctly.

11. Ordered vs Unordered Lists

+

Below is a quick comparison between ordered lists and unordered lists.

Feature Ordered List ( <ol> ) Unordered List ( <ul> )
Purpose Used when the order or sequence of items is important. Used when order does not matter.
Appearance Numbers (1, 2, 3...), letters (a, b), or roman numerals (i, ii). Bullets (●, ○, ■).
Syntax <ol><li>Item</li></ol> <ul><li>Item</li></ul>
Example Usage Steps, tutorials, processes, rankings. Features list, menu items, categories.
Browser Default Numeric numbering. Solid bullet.
Customization The type attribute can change number style. The list-style property can change bullet style.

14. Self-Closing Tags

+

Self‑closing tags do not require a separate closing element. They represent elements that do not contain inner content.

Common examples include <br> for line breaks, <hr> for horizontal rules, <img> for images, <input> for form fields, <meta> for metadata, and <link> for connecting external CSS files.

Example :

<img src="pic.jpg" alt="Photo">
<br>

Self‑closing tags are short, clear, and important parts of HTML syntax and structure.

15. Links (Hyperlinks)

+

Links connect webpages internally (within the same site) and externally (to other websites).

Example :

<a href="https://google.com" target="_blank" title="Open Google">Visit Google</a>

Important Attributes :

href – the link address (URL)
target="_blank" – opens the link in a new tab
title – shows hover text / tooltip on the link

16. Images (With Attributes)

+

Images make webpages more engaging. The <img> tag is self‑closing and uses attributes to define the source, description, and size.

Example :

<img src="laptop.png" alt="Laptop Image" width="300">

The alt text is important for:

  • Accessibility – screen readers can describe the image to visually impaired users.
  • SEO – search engines understand the image content.
  • Image loading failures – text is shown when the image cannot load.

17. Semantic HTML

+

Semantic HTML uses meaningful tags so that the structure of the page clearly describes the content.

Common Semantic Tags :

  • <header> – top section (logo, title, main navigation)
  • <nav> – navigation menu with links
  • <section> – groups related content
  • <article> – independent content (blog post, news article)

18. Forms & Input Types

+

The <form> tag collects user input and sends it to a server or processes it using JavaScript. Forms can contain text fields, radio buttons, checkboxes, dropdowns, buttons, and password fields. Common input types include text , email , password , number , and submit . Forms are essential for login pages, registrations, surveys, search boxes, and payment forms.

✔ Example Form

<form>
  <label>Name:</label>
  <input type="text">

  <label>Email:</label>
  <input type="email" required>

  <button>Submit</button>
</form>

✔ Common Input Types

Some commonly used input types are:

  • text
  • password
  • email
  • number
  • radio
  • checkbox
  • date
  • file

✔ Validation Attributes

Validation attributes help control and verify the data users enter before the form is submitted.

  • required – makes the field mandatory
  • min – sets the minimum numeric value
  • max – sets the maximum numeric value
  • pattern – defines a regular expression that the input must match (useful for custom formats)
  • placeholder – shows hint text inside the field explaining what to enter

19. Tables (With Proper Structure)

+

Tables are used to display structured, tabular data like products, prices, marks, or schedules.

✔ Example :

<table border="1">
  <thead>
    <tr>
      <th>Product</th>
      <th>Price</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>Keyboard</td>
      <td>₹800</td>
    </tr>
  </tbody>
</table>

Tables should be used only for data, not for page layout. For layout, prefer CSS with flexbox or grid.
