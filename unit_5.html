Unit 5 — DOM, Events & User Interaction

1. Introduction to DOM (Document Object Model)

+

The DOM is a programming interface that represents the webpage as a tree-like structure where every element (tags, attributes, and text) becomes a node . JavaScript can access, read, update, delete, or create elements using the DOM. It allows developers to make web pages interactive — such as showing messages, hiding elements, changing styles, validating forms, or responding to user input.

The DOM is created by the browser when it reads the HTML file.

✔ Diagram of DOM Tree (Text View)

Document
└── html
    ├── head
    └── body
        ├── h1
        ├── p
        └── button

2. Accessing DOM Elements

+

We use JavaScript methods to select HTML elements.

Common DOM Selectors

Method Description
document.getElementById() Selects element by id
document.getElementsByClassName() Selects elements by class
document.getElementsByTagName() Selects elements by tag
document.querySelector() Selects first matching element
document.querySelectorAll() Selects all matching elements

Example

HTML

<h1 id="title">Hello</h1>
<p class="msg">Welcome!</p>

JavaScript

let heading = document.getElementById("title");
let text = document.querySelector(".msg");

console.log(heading.innerText);
console.log(text.innerText);

3. Manipulating DOM Elements (Changing Content, Style, Attributes)

+

DOM manipulation means modifying the webpage dynamically using JavaScript.

Change Text

document.getElementById("title").innerText = "Updated Title";

Change HTML

document.getElementById("box").innerHTML = "<b>Bold Text</b>";

Change CSS Style

document.getElementById("title").style.color = "blue";
document.getElementById("title").style.fontSize = "30px";

Change Attributes

document.getElementById("img").src = "photo.png";
document.getElementById("btn").disabled = true;

Add/Remove Classes

element.classList.add("active");
element.classList.remove("hidden");

4. Events in JavaScript (Very Important)

+

Events are actions that happen on a webpage (clicking, typing, scrolling). JavaScript allows you to respond to these actions.

✔ Common Events

Event Fired When
click User clicks a button
mouseover Mouse enters element
mouseout Mouse leaves element
keyup Key is released
keydown Key is pressed
submit Form is submitted
change Input value changes

5. Adding Events (3 Methods)

+

Method 1: Inline Event (HTML)

<button onclick="greet()">Click</button>

<script>
function greet() {
  alert("Hello!");
}
</script>

Method 2: Using DOM Property

let btn = document.getElementById("btn");
btn.onclick = function () {
  alert("Button Clicked!");
};

Method 3: addEventListener() ⭐ (MOST IMPORTANT)

btn.addEventListener("click", () => {
  alert("Button Pressed");
});

Why addEventListener() ?

  • ✔ Allows multiple events on same element
  • ✔ Cleaner and easier to manage
  • ✔ Supports advanced features (like removing listeners, options)

6. Responding to User Input (Forms & Buttons)

+

JavaScript reads user input using the .value property.

Example: Getting Input Value

HTML:

<input type="text" id="nameInput">
<button id="btn">Submit</button>
<p id="output"></p>

JavaScript:

document.getElementById("btn").addEventListener("click", function () {
  let name = document.getElementById("nameInput").value;
  document.getElementById("output").innerText = "Hello " + name;
});

7. Basic Form Validation (Exam Important)

+

Validation checks if the form input is correct before submitting.

Example: Required Field Check

let email = document.getElementById("email").value;

if (email === "") {
  alert("Email is required");
}

Example: Check Minimum Length

let pass = document.getElementById("pass").value;

if (pass.length < 6) {
  alert("Password must be at least 6 characters");
}

Example: Validate Number Input

if (isNaN(age)) {
  alert("Enter valid number");
}

11. Practical Example — Form Validation + DOM Update

+

This example combines form validation and DOM update . We check if an email is valid and then update a <p> element's text and color based on the result.

HTML:

<input id="email" placeholder="Enter email">
<button id="check">Validate</button>
<p id="result"></p>

JavaScript:

check.addEventListener("click", () => {
  let mail = email.value;

  if (mail.includes("@")) {
    result.innerText = "Valid Email";
    result.style.color = "green";
  } else {
    result.innerText = "Invalid Email";
    result.style.color = "red";
  }
});

Explanation:

  • The button with id check listens for a click event.
  • On click, we read the value from the input with id email into variable mail .
  • We do a simple check using includes("@") to see if the email contains @ .
  • If valid, we update the paragraph with id result to show "Valid Email" in green color.
  • If invalid, we show "Invalid Email" in red color.
  • This shows how events, validation, and DOM styling work together in a real mini-project.

12. Notes for "Recreate Layout Using DOM" Questions

+

In exams you may be asked to create a layout using JavaScript + DOM instead of writing plain HTML. Students must know how to:

// 1. Create element
const div = document.createElement("div");

// 2. Set text
div.innerText = "Hello DOM";

// 3. Add attributes (id, class)
div.id = "box";
div.setAttribute("class", "card");

// 4. Apply styles
div.style.backgroundColor = "lightblue";
div.style.padding = "10px";

// 5. Append into page
document.body.appendChild(div);

// 6. Attach event
div.addEventListener("click", () => {
  alert("Div clicked!");
});

Tip for answers: write the steps clearly in order (Create → Set text/attributes → Style → Append → Add events). This shows full understanding of building UI with the DOM.
