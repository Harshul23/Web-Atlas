Fundamentals

1. Introduction to JavaScript

-

JavaScript is a high-level programming language used to add interactivity and dynamic behavior to websites. It runs directly inside the browser, allowing developers to respond to user actions such as clicking, typing, scrolling, and submitting forms. JavaScript interacts with the browser through the DOM (Document Object Model), enabling changes to HTML and CSS in real time.

JavaScript supports variables, functions, loops, events, arrays, and objects. It is essential for modern web development and is also used in backend (Node.js), mobile apps, and desktop applications.

2. JavaScript Data Types

+

JavaScript has different types of data used to store values in variables.

Primitive Types :

  • String – text
  • Number – integer or decimal
  • Boolean – true / false
  • Undefined – declared but not assigned
  • Null – empty or intentional absence
  • Symbol – unique values
  • BigInt – very large numbers

Reference Type :

  • Object – collection of key–value pairs

Example :

let name = "Alice";      // String
let age = 21;            // Number
let isStudent = true;    // Boolean

let data = null;         // Null
let user;                // Undefined

let person = {           // Object
  name: "Bob",
  age: 25
};

3. Variables: var, let, and const

+

In JavaScript, var , let , and const are used to declare variables, but they differ in scope, hoisting, and whether you can re-declare or re-assign them.

Quick Comparison :

  • Scope : var is function-scoped , while let and const are block-scoped (limited to {} blocks).
  • Hoisting : all are hoisted, but var is initialized as undefined , whereas let and const stay in the Temporal Dead Zone (TDZ) until their line of initialization.
  • Re-declaration : var can be re-declared in the same scope; let and const cannot.
  • Re-assignment : var and let can be re-assigned; const cannot be re-assigned after the first value.
  • Default initialization : var defaults to undefined ; let and const have no default value and throw if used before initialization (TDZ).
  • Global object property : in browsers, a global var becomes a property on window (for example, window.varName ), whereas let and const do not.

Use Cases & Best Practice :

  • var : for older JS code or when function-level scope is specifically required. Best practice: generally avoid in modern code.
  • let : for variables whose values need to change over time (loops, counters, temporary values). Best practice: recommended for most variables.
  • const : for values that should not be re-assigned (configuration values, references to arrays/objects). Best practice: use by default when you do not need to re-assign.

Small Example :

var x = 10;      // function-scoped
let y = 20;      // block-scoped
const PI = 3.14; // block-scoped, cannot be re-assigned

if (true) {
  let y = 30;   // different 'y' (inside block)
}

// PI = 3.1415; // Error: Assignment to constant variable

5. Hoisting in JavaScript

+

Hoisting is JavaScript's behavior of moving declarations to the top of their scope before the code actually runs. Because of this, you can sometimes use variables and functions before they are written in the code.

However, different declarations behave differently when hoisted:

  • var → hoisted and initialized as undefined .
  • let / const → hoisted but not initialized (they stay in TDZ).
  • Function declarations → fully hoisted, so you can call them before their definition.

Example :

console.log(a); // undefined (var is hoisted and initialized)
var a = 10;

The declaration var a is moved to the top of its scope and set to undefined , so the first console.log does not throw an error.

6. Temporal Dead Zone (TDZ)

+

The Temporal Dead Zone (TDZ) is the time between when a let or const variable is hoisted and when it is initialized with a value . During this period, the variable exists in memory but cannot be accessed — any access will throw a ReferenceError .

This explains why let and const are safer than var : they prevent you from accidentally using a variable before it is properly initialized.

Example :

console.log(x); // ReferenceError: Cannot access 'x' before initialization
let x = 20;

Here x is hoisted but stays in the TDZ until the let x = 20; line executes. Accessing it earlier causes a ReferenceError .

Quick View: var vs let/const (Hoisting & TDZ)

Feature var let / const
Hoisted? Yes, declaration moved to top. Yes, declaration also moved to top.
Initial value before line undefined No value (TDZ → cannot access)
Access before initialization Allowed, gives undefined . Not allowed, throws ReferenceError .
Safety Can hide bugs (silent undefined values). Safer — forces correct order of code.

4. Introduction to Loops in JavaScript

+

In JavaScript, loops are used to repeat a block of code multiple times. They help avoid writing the same code repeatedly. For example, if you want to print numbers from 1 to 100, loops allow you to do this with just a few lines of code.

Loops are essential in array processing, pattern generation, calculations, animations, and DOM manipulations. JavaScript provides different kinds of loops, such as for , while , do-while , and enhanced loops like for...of and for...in . Each loop has specific use cases: some are better when the number of iterations is known, while others are suitable when the iterations depend on a condition.

1. For Loop

The for loop is the most commonly used loop in JavaScript. It is ideal when the number of iterations is already known. It has three parts: initialization , condition , and increment / decrement . The loop runs as long as the condition is true.

Syntax Diagram :

for (initialization; condition; increment/decrement) {
  // code runs repeatedly
}

Example :

for (let i = 1; i <= 5; i++) {
  console.log(i);
}

2. While Loop

The while loop runs as long as the condition remains true. It is preferred when the number of iterations is unknown. The body of the loop may execute zero or more times depending on the condition.

Example :

let i = 1;

while (i <= 5) {
  console.log(i);
  i++;
}

3. Do-While Loop

The do-while loop executes its block at least once before checking the condition. This makes it suitable when you must guarantee a minimum of one execution.

Example :

let i = 1;

do {
  console.log(i);
  i++;
} while (i <= 5);

5. Difference Between For and While Loop

+

The for loop and while loop can often solve the same problems, but they are suited to slightly different situations. The table below summarizes the key differences.

Comparison Table :

Feature For Loop While Loop
Iteration Count Known before starting in most cases. May be unknown and depends on a condition.
Syntax Compact — initialization, condition, and increment in one line. Condition checked separately; initialization / increment usually written outside or inside the body.
Use Case Arrays, counting loops, fixed number of iterations. Waiting for a condition to become false; condition-driven loops.
Risk Less chance of infinite loop because increment is usually visible in the header. Higher chance of infinite loop if the condition or increment is not updated.

6. for...of Loop

+

The for...of loop is used to iterate through values of iterable objects like arrays, strings, maps, and sets. It provides a clean and easy way to access elements without worrying about indexes.

It is mostly used when you only need values and not their positions. This loop is commonly used in array processing, form validation, string character checks, and modern JavaScript frameworks. It reduces errors and improves readability compared to traditional for loops.

Example :

let nums = [10, 20, 30];

for (let n of nums) {
  console.log(n);
}

7. for...in Loop

+

The for...in loop is used to iterate over keys or property names of an object . It is ideal when working with key–value structures such as user profiles, configurations, student records, or JSON data. While it can also loop through array indexes, it is recommended mainly for objects, not arrays.

This loop helps inspect and process object data easily, especially when you want to access both the property name (key) and its corresponding value.

Example :

let student = { name: "Mitali", age: 21, city: "Pune" };

for (let key in student) {
  console.log(key, student[key]);
}

8. Nested Loops

+

Nested loops are loops inside other loops . They are used for matrix operations, patterns, tables, and multi-level iteration where one loop depends on another.

In nested loops, the inner loop runs completely for every single iteration of the outer loop . This can lead to many total iterations, so they should be used carefully to avoid performance issues.

Example :

for (let i = 1; i <= 3; i++) {
  for (let j = 1; j <= 3; j++) {
    console.log(i, j);
  }
}

9. Conditional Statements

+

Conditional statements in JavaScript allow the program to make decisions based on conditions . They control the flow of execution depending on whether an expression evaluates to true or false . This is a fundamental concept because almost every real-world program requires checking conditions — for example, age verification, login validation, marks evaluation, access permissions, pricing logic, and more.

JavaScript provides multiple conditional structures like if , if-else , else-if ladder , ternary operator , and switch . Each one is suitable for a different type of decision-making. Understanding conditional statements helps build logical thinking and real-world application programs.

1. if Statement

The if statement is the simplest decision-making structure in JavaScript. It checks a condition and executes a block of code only if that condition evaluates to true . If the condition is false , the block is skipped. It is useful when you want to run something only under one specific scenario — for example, checking if a user is old enough to vote, verifying if temperature is too high, or validating if an input field is not empty.

Example :

let age = 18;

if (age >= 18) {
  console.log("You are an adult");
}

2. if–else Statement

An if-else statement provides two possible paths: one for the true condition and one for the false condition. This is helpful in scenarios where you must choose between two outcomes — such as pass/fail, login success/error, or discount eligibility. If the condition is true, the first block runs; otherwise, the else block executes. This ensures that the program always performs one action between the two choices.

Example :

let marks = 45;

if (marks >= 40) {
  console.log("Pass");
} else {
  console.log("Fail");
}

3. else-if Ladder

The else-if ladder is used when there are more than two possible outcomes. It checks conditions in sequence, and the first true condition executes its code block while skipping the rest. This structure is useful for multi-level decisions like grading systems, temperature categories, menu selections, or login roles (admin, user, guest). It makes code more organized and avoids writing multiple separate if statements.

Example :

let temp = 30;

if (temp > 40) {
  console.log("Very Hot");
} else if (temp > 25) {
  console.log("Warm");
} else {
  console.log("Cold");
}

4. Ternary Operator

The ternary operator is a shorthand form of an if-else statement. It is mostly used for simple decision-making where a value needs to be returned or assigned based on a condition. Because it takes only one line, it is widely used in UI logic, inline conditions, and quick value assignments. However, it should not be used for very complex decisions because that makes code harder to read.

Syntax :

condition ? valueIfTrue : valueIfFalse

Example :

let age = 20;

let status = age >= 18 ? "Adult" : "Minor";
console.log(status);

5. switch Statement

The switch statement is used for making decisions when you want to compare a variable against multiple fixed values. It is cleaner than using many else-if blocks. Each case is matched with the variable, and when a match is found, that block executes. It is especially useful for menu selection, days of the week, grade mapping, or fixed-input conditions. Adding break statements prevents fall-through.

Example :

let day = 3;

switch (day) {
  case 1:
    console.log("Mon");
    break;
  case 2:
    console.log("Tue");
    break;
  case 3:
    console.log("Wed");
    break;
  default:
    console.log("Invalid day");
}

6. Decision-Making Flowchart

A decision-making flowchart helps visualize how conditional statements work in a logical flow. It shows how a program checks a condition and chooses one of the available paths. This is especially important for beginners because it helps them see how the code executes step-by-step. Flowcharts make troubleshooting easier because you can spot where your logic branches incorrectly. Such diagrams are commonly used in software design, debugging, and planning algorithms.

a. Nested If Statements

Nested if statements are multiple if-else conditions placed inside one another. They are used when you need to check multiple conditions in a structured manner, where each inner condition depends on the previous condition being true . Nested if is common in grading systems, login validation, and user input checking. However, too many nested conditions can make the code hard to read, so indentation and formatting are important.

Example :

let marks = 82;

if (marks >= 90) {
  console.log("Grade: A+");
} else if (marks >= 75) {
  if (marks >= 85) {
    console.log("Grade: A (high)");
  } else {
    console.log("Grade: A");
  }
} else if (marks >= 60) {
  console.log("Grade: B");
} else {
  console.log("Grade: C or below");
}

10. Operators & Expressions

+

JavaScript supports different types of operators such as arithmetic ( + , - , * , / ), comparison ( == , === , < , > ), and logical operators ( && , || , ! ). These operators are used inside expressions to perform calculations, compare values, and combine conditions.

An expression is any valid combination of variables, values, and operators that produces a result. For example, (5 > 3) && (10 > 2) is a logical expression that evaluates to true because both comparison conditions are true.

Example :

let result = (5 > 3) && (10 > 2);

console.log(result); // true

Functions


11. Functions: What is a Function in JavaScript?

+

A function in JavaScript is a reusable block of code designed to perform a specific task. Instead of writing the same code again and again, functions let us write logic once and reuse it many times. Functions improve structure, reduce repetition, and make code easier to maintain. A function can take inputs (called parameters ) and may return an output using the return keyword.

In JavaScript, functions are "first-class citizens" , meaning they can be stored in variables, passed as arguments, and returned from other functions. Functions are the foundation for many programming concepts such as event handling, callbacks, array methods ( map , filter , reduce ), API requests, and component-based development.

a. Function Syntax

Basic Structure

function functionName(parameter1, parameter2) {
  // code to execute
  return value;
}

b. Diagram of Function Flow

Input (Arguments) ↓ Function Definition ↓ Processing / Logic ↓ Output (Return Value)

Example :

function greet() {
  console.log("Hello, welcome!");
}

greet();

(i) Function With Return Value

A function can return a value back to the place where it was called. After the return statement runs, the function stops executing and sends the result.

Example :

function square(num) {
  return num * num;
}

console.log(square(6)); // 36

The function accepts a parameter num , processes it, and returns the computed result using return . After returning, the function stops execution.

(ii). Function Parameters vs Arguments

  • Parameters — variables listed in the function definition.
  • Arguments — actual values passed when calling the function.

Example :

function add(a, b) {  // a, b → parameters
  return a + b;
}

add(5, 3);   // 5, 3 → arguments

(iv). Why Do We Use Functions?

Functions are used to break code into smaller logical units. They allow us to avoid repetition through reusability, which increases efficiency and reduces bugs. Functions help organize code so it is modular, clean, and easier to read. In larger projects, functions serve as building blocks for features like form validation, calculations, event handling (onClick, onHover), data processing, and even UI components.

Many JavaScript libraries and frameworks (such as React and Node.js) rely heavily on functions as their primary structure for logic. Thus, without functions, writing scalable and maintainable code becomes almost impossible.

12. Types of Functions in JavaScript

+

(i). Named Function

function hello() {
  console.log("Hello");
}

(ii). Function Expression

const add = function(a, b) {
  return a + b;
};

(iii). Arrow Functions in JavaScript (ES6)

Arrow functions were introduced in ES6 to provide a shorter and cleaner way to write functions. They reduce the amount of code and improve readability. Unlike regular functions, arrow functions do not have their own this , arguments , or prototype , making them useful for callbacks, event handlers, and array methods ( map , filter , reduce ) and functional programming.

Arrow functions always use the => ("fat arrow") syntax and can be written in multiple forms depending on whether they return a single expression or require multiple statements. They are especially useful when working with small, simple functions because of their concise syntax and help avoid unnecessary boilerplate code.

✔ Arrow Function Syntax

const functionName = (parameters) => {
  // function body
};

a. Simple Arrow Function (Implicit Return)

If the function has only one expression, you can remove the {} and the return keyword. The expression value is returned automatically.

Example :

const multiply = (a, b) => a * b;

console.log(multiply(4, 5)); // 20

b. Arrow Function With Multiple Statements

When the function body has more than one statement, use {} and include an explicit return statement.

Example :

const addNumbers = (a, b) => {
  let sum = a + b;
  return sum;
};

c. Arrow Function Without Parameters

When there are no input values, use empty parentheses before the => arrow.

Example :

const greet = () => console.log("Hello!");

d. Arrow Function With One Parameter

If there is exactly one parameter, the parentheses around it are optional.

Example :

const square = n => n * n;

13. Difference Between Normal Function & Arrow Function

+

Normal (regular) functions and arrow functions both allow us to define reusable blocks of code, but they behave differently in terms of syntax, this binding, availability of the arguments object, and where they are best used.

Feature Comparison :

Feature Normal Function Arrow Function
Syntax Longer (uses the function keyword). Shorter (uses the => arrow syntax).
this binding Has its own this (depends on how it is called). Inherits this from surrounding scope (lexical this ).
arguments object Available inside the function. Not available (use rest parameters instead).
Suitable for Object methods, constructor functions. Callbacks, array methods ( map , filter , reduce ).

Example Showing This Difference

In the example below, the normal function correctly uses this.name from the object, but the arrow function does not get its own this and therefore does not work as expected. 

const person = {
  name: "Mitali",
  greet: function() {
    console.log("Hello " + this.name); // Works
  },
  greetArrow: () => {
    console.log("Hello " + this.name); // ❌ undefined (no own this)
  }
};

person.greet();
person.greetArrow();

14. Nested Functions (Function Inside Function)

+

JavaScript allows defining one function inside another function . Such functions are called nested functions . The outer function contains the inner function and can call it inside its body.

The important point is that the inner function has access to the variables of the outer function . This is related to the concept of closures , where a function remembers the scope in which it was created.

Example :

function outer() {
  let a = 5;

  function inner() {
    console.log(a); // inner can access outer variables
  }

  inner();
}

outer();

Arrays


14. Arrays: Introduction & Indexing

+

1. What is an Array?

An array in JavaScript is a data structure used to store multiple values in a single variable . Instead of creating many separate variables, arrays let you store lists of numbers, strings, objects, and more. Arrays are ordered , meaning each value has an index starting from 0 .

JavaScript arrays are dynamic , so you can add or remove elements at any time. Arrays are widely used in loops, data processing, API responses, and UI rendering.

Example :

let numbers = [10, 20, 30, 40];
console.log(numbers[0]); // 10

2. Array Indexing

Each element in an array has a position called its index . Indexing in JavaScript arrays always starts from 0 .

Element 10 20 30 40
Index 0 1 2 3

To access an element, use the array name followed by the index in square brackets.

Access Example :

numbers[2]; // 30

15. Common Array Methods

+

JavaScript arrays provide many built-in methods to add, remove, and search elements. Below are some of the most commonly used methods in day-to-day programming.

push() – Add element at end

Adds one or more elements to the end of an array and returns the new length. 

let nums = [1, 2, 3];
nums.push(4);
console.log(nums); // [1, 2, 3, 4]

pop() – Remove element from end

Removes and returns the last element of the array. 

let nums = [1, 2, 3];
nums.pop();
console.log(nums); // [1, 2]

shift() – Remove first element

Removes and returns the first element of the array. 

let items = ["a", "b", "c"];
items.shift();
console.log(items); // ["b", "c"]

unshift() – Add element at start

Adds one or more elements to the beginning of the array and returns the new length. 

let items = ["b", "c"];
items.unshift("a");
console.log(items); // ["a", "b", "c"]

slice() – Extract a portion

Returns a shallow copy of a portion of an array into a new array. It does not change the original array. 

let arr = [10, 20, 30, 40];
let part = arr.slice(1, 3);
console.log(part); // [20, 30]

indexOf() – Find position of value

Returns the index of the first occurrence of a value in the array, or -1 if not found. 

let fruits = ["apple", "banana", "orange"];
let index = fruits.indexOf("banana");
console.log(index); // 1

16. Advanced Array Methods (Internal Looping)

+

JavaScript provides special array methods such as map() , filter() , and reduce() , which internally use loops to process elements. These methods help avoid writing manual for loops , making code cleaner, shorter, and easier to understand.

Although the code looks simple when using these methods, JavaScript actually performs hidden loop operations behind the scenes .

1. map() — Transform Every Element

Purpose

map() iterates through each element of an array and returns a new array where every value is transformed based on the given function (callback).

Internal Working (concept)

You can imagine map() doing this internally:

  • for each item in the array:
  • apply callback function
  • store the transformed value in a new array
  • return new array

Key Points

  • Does not modify the original array.
  • Always returns a new array of same length .
  • Used for: doubling values, formatting strings, converting data, etc.

Example :

let numbers = [2, 5, 8, 10, 13];
let doubled = numbers.map((n) => n * 2);
console.log(doubled); // [4, 10, 16, 20, 26]

Explanation

  • Each value n is multiplied by 2 .
  • JavaScript performs the looping behind the scenes .
  • A new array is created containing all the modified values.

2. filter() — Select Matching Elements

Purpose

filter() returns a new array containing only the elements that satisfy a condition (true/false check).

Internal Working (concept)

You can imagine filter() doing this internally:

  • for each item in the array:
  • if callback returns true :
  • push element to new array
  • return filtered array

Key Points

  • Does not change the original array.
  • Returns a shorter (or equal) array.
  • Used for: selecting even numbers, filtering students by marks, removing duplicates, etc.

Example :

let numbers = [2, 5, 8, 10, 13];
let even = numbers.filter((n) => n % 2 === 0);
console.log(even); // [2, 8, 10]

Explanation

  • Condition checks if the number is divisible by 2 .
  • Only even values are included in the new array.
  • Internally, JavaScript loops through each value and performs the test.

3. reduce() — Convert Array into a Single Value

Purpose

reduce() applies a function to all elements of an array and combines them into one final result (such as sum, product, total marks, etc.).

Internal Working (concept)

You can imagine reduce() doing this internally:

  • set an accumulator to the initial value (for example, 0)
  • for each item in the array:
  • accumulator = callback(accumulator, currentElement)
  • return the final accumulator

Key Points

  • Can generate numbers, strings, or even objects.
  • Very powerful method, used in many real-world cases.
  • Used for: total marks, total price, flattening arrays, counting frequency, etc.

Example : Sum of All Values

let numbers = [2, 5, 8, 10, 13];
let sum = numbers.reduce((acc, n) => acc + n, 0);
console.log(sum); // 38

Explanation

  • acc starts at 0 (initial value).
  • Each number n is added to acc one by one.
  • Loop continues until the array is finished, and the final result is 38 .

Visual Diagram of How These Methods Work

Original Array : [2, 5, 8, 10, 13]

  • map() [4, 10, 16, 20, 26]
  • filter() [2, 8, 10]
  • reduce() 38

Each method loops internally but returns different types of results :

Method Loop Output Returned Value
map() transformed values new array (same length)
filter() condition check new array (shorter)
reduce() combine values single value (number/string/object)

Final Example (All Three Together)

var numbers = [2, 5, 8, 10, 13];
let doubled = numbers.map(n => n * 2);   // transforms
let even = numbers.filter(n => n % 2 === 0); // selects
let sum = numbers.reduce((acc, n) => acc + n, 0);  // combines

console.log(doubled); // [4, 10, 16, 20, 26]
console.log(even);    // [2, 8, 10]
console.log(sum);     // 38

Bonus: forEach() and find()

forEach() — Loop through elements

The forEach() method runs a given function once for every element in the array. It is mainly used when you want to perform an action for each value (like printing, updating DOM, or multiplying) but do not need a new array as the result.

You can imagine forEach() as a loop that automatically moves through the array and passes each element to your callback function. 

let arr = [2, 3, 4];

arr.forEach(n => console.log(n));

Example: Multiply All Values

let arr = [2, 3, 4];
let product = 1;

arr.forEach(n => {
  product *= n;
});

console.log(product); // 24

Here the callback function multiplies the product variable by each value of n in the array. After the loop finishes, we get the final multiplication result.

find() — Returns first matching value

let nums = [1, 3, 6, 8];
let f = nums.find(n => n > 5);
console.log(f); // 6

Object


17. Object — What is an Object?

+

Objects in JavaScript are used to store data in key–value pairs . They represent real-world entities such as students, users, products, cars, and orders. Each property has a key (name) and a value (data).

Objects allow grouping related data together, making it easier to manage complex structured information. JavaScript heavily uses objects — arrays, functions, and even the browser's DOM are all objects internally.

Example :

let student = {
  name: "Mitali",
  rollNumber: 21,
  marks: 88
};

18. Object — Accessing Object Values

+

There are two main ways to read values from an object: dot notation and bracket notation .

1. Dot Notation

Used when you know the exact property name and it is a valid identifier (no spaces or special characters).

console.log(student.name);       // "Mitali"
console.log(student.marks);      // 88

2. Bracket Notation

Used when the property name is stored in a variable or contains spaces / special characters. The key is written as a string inside square brackets. 

console.log(student["marks"]);    // 88

let key = "name";
console.log(student[key]);      // "Mitali"

19. Object — Adding, Updating & Deleting Properties

+

Objects are mutable , which means you can change their properties at any time — add new ones, update existing ones, or delete properties completely.

1. Adding or Updating Properties

Use dot or bracket notation with the = operator. If the key already exists, its value is updated ; if it does not exist, a new key is added . 

let student = {
  name: "Mitali",
  rollNumber: 21,
  marks: 88
};

student.marks = 92;        // update existing property
student.city = "Pune";   // add new property

2. Deleting a Property

Use the delete operator to remove a property completely from the object. 

delete student.rollNumber;

20. Complete Program & Looping with for...in

+

1. Object Program (Display Details)

let student = {
  name: "Mitali",
  rollNumber: 21,
  marks: 88
};

console.log("Name:", student.name);
console.log("Roll:", student.rollNumber);
console.log("Marks:", student.marks);

2. Looping Through Objects — for...in

The for...in loop is used to iterate over the keys (property names) of an object. For each key, you can access the corresponding value using bracket notation. 

let user = { name: "Mitali", age: 21 };

for (let key in user) {
  console.log(key, user[key]);
}

How this loop works (step-by-step)

  • for...in picks each property name (key) from the user object one by one.
  • In every iteration, the key variable holds a different property name like "name" or "age" .
  • We use user[key] (bracket notation) to read the value of that property.
  • This pattern is very useful when you need to inspect or print all properties of an object without knowing the property names in advance.

21. Array of Objects

+

An array of objects is commonly used to store lists of similar records such as students, products, or users. Each element of the array is its own object with properties.

This pattern is very common in APIs, databases, and tables where you need to handle multiple rows of structured data.

Example :

let students = [
  { name: "Mitali", marks: 90 },
  { name: "Riya",   marks: 85 },
  { name: "Amit",   marks: 78 }

console.log(students[1].name); // Riya

22. null vs undefined (Object Related)

+

Both null and undefined represent "no value" , but they are used in slightly different situations.

Feature null undefined
Meaning intentional empty value not assigned
Type object undefined
Set by developer JavaScript
Use reset / clear values missing or not initialized values

In object logic you often set a property to null when you know it exists but want to mark it as empty for now . If a property is undefined , it usually means it was never assigned or the key does not exist.

23. Object — == vs === inside Object Logic

+

JavaScript has two equality operators: == (loose) and === (strict). When working with objects and conditions, using === avoids unexpected type conversions.

Example :

console.log(5 == "5");   // true  (values are equal after conversion)
console.log(5 === "5");  // false (type number vs string)

In object checks (for example comparing IDs inside objects), prefer === so that 5 and "5" are treated as different values unless you intentionally want automatic type conversion.
