Unit 4 — Git & Version Control

1. Introduction to Version Control (VCS)

+

Version Control Systems (VCS) help developers manage changes made to code over time. They allow tracking of modifications, restoring previous versions, and collaborating with multiple developers without overwriting each other’s work. VCS also maintains a complete history of changes, making debugging and team coordination easier. Git is a distributed VCS, meaning every user has a full copy of the project history, ensuring speed, reliability, and offline access.

Key Features

  • Tracks every change
  • Provides backups
  • Allows collaboration
  • Helps revert mistakes
  • Maintains version history

2. What is Git?

+

Git is a distributed version control system used to track changes in source code. Each contributor has a full copy of the repository, including the complete history. Git is fast, efficient, and widely used in professional development. It helps manage projects, work in teams, resolve conflicts, and maintain multiple versions of a project through branches.

3. Git Installation & Setup

+

Before using Git, you need to install it on your system and configure your identity (name and email). This information is stored with every commit you make.

Install Git

Download Git from the official website: https://git-scm.com/downloads

Set Username & Email (important for commits)

These details are used to identify the author of each commit.

git config --global user.name "Your Name"
git config --global user.email "your@email.com"

Check Configuration

To verify your Git configuration, run:

git config --list

4. Repository Creation (Local Repo)

+

A Git repository tracks changes in a project. You can turn any folder into a Git repository and start tracking all version history.

Creating a New Repository

Inside your project folder, run:

git init

This creates a hidden .git folder which stores all version history (commits, branches, etc.).

Clone an Existing Repository

To copy an existing remote repository to your system, use:

git clone <repository-url>

6. Tracking Files in Git: Add → Commit → Push

+

When working with Git, you generally follow these basic steps to record and share your work:

1. Check Status

git status

2. Stage Files

Stage individual files:

git add file.html

Or stage everything in the current folder:

git add .

3. Commit Changes

git commit -m "Meaningful message"

4. Push to GitHub

git push origin main

7. Understanding Staging & Committing

+

Staging is the process of selecting which modified files should be included in the next commit. This allows developers to organize changes and commit them in meaningful groups.

Committing stores the staged files into the repository with a message describing the change. Each commit becomes a snapshot of the project, which can be restored anytime.

8. Using GitHub for Remote Collaboration

+

GitHub is a cloud platform for hosting Git repositories. It enables developers to collaborate, share code, review changes, and contribute to the same project from anywhere.

Key Features

  • Remote backups
  • Pull requests
  • Issues & bug tracking
  • Code reviews
  • Team collaboration

Connect Git with GitHub

To link your local repository with a remote on GitHub, use:

git remote add origin <repo-url>

9. Pushing & Pulling Code (Remote Workflows)

+

Push Code to GitHub

git push origin main

Pull Latest Changes from GitHub

git pull origin main

Fetch Only (without merging)

git fetch

10. Understanding Branches

+

A branch is a separate version of your project where you can add features without affecting the main code. Branches help developers experiment safely.

Common Commands (Intro Level)

Create a new branch:

git branch feature1

Switch branch:

git checkout feature1

List all branches:

git branch

Merge a branch into main:

git merge feature1

11. Important Git Commands (Theory + Examples)

+

1. git init — Initialize a Repository

git init is used to create a new Git repository inside a folder. It starts version control by generating a hidden .git folder that tracks file changes, commits, and branches.

Example

git init

2. git add — Move Changes to Staging Area

git add moves modified files from the working directory to the staging area. This step prepares changes for commit so Git only records what is staged, giving full control over versioning.

Example

git add file.html
git add .

3. git commit — Save a Version Snapshot

git commit permanently records the staged changes into the repository history. It creates a snapshot of your project at a specific point in time along with a meaningful commit message.

Example

git commit -m "Added homepage UI"

4. git status — Check Current Repo State

git status shows which files are modified, staged, or untracked. It helps developers monitor what has changed and decide what needs to be added or committed.

Example

git status

5. git branch — Manage Branches

git branch is used to create, list, or delete branches. It lets developers work on new features without changing the main project, making collaboration safer and easier.

Example

git branch
git branch new-feature

6. git checkout — Switch Branches

git checkout is used to move between branches or restore files. It allows developers to switch contexts instantly and work on different versions of the project.

Example

git checkout new-feature

7. git merge — Combine Branches

git merge combines changes from one branch into another. It integrates work done by different developers, and if the same file was edited differently, merge conflicts may occur.

Example

git merge new-feature

8. git clone — Copy Remote Repository

git clone creates a full copy of a remote GitHub repository on your local machine. It downloads all files, history, branches, and automatically connects to the remote origin.

Example

git clone https://github.com/user/project.git

9. git remote add origin — Connect Local Repo to GitHub

git remote add origin links your local project to a remote GitHub repository so you can push and pull code between them.

Example

git remote add origin https://github.com/user/repo.git

10. git push — Upload Code to GitHub

git push sends your local commits to the remote repository (GitHub). This is how developers share their code and collaborate with team members working on the same project.

Example

git push -u origin main

11. git pull — Download Updates from GitHub

git pull fetches the latest changes from the remote repository and automatically merges them into your local code. It keeps your project synchronized with teammates’ updates.

Example

git pull origin main

12. git fetch — Download Without Merging

git fetch downloads new commits from the remote but does not merge them automatically. It is safe for reviewing updates before changing your local branches.

Example

git fetch origin

13. git log — View Commit History

git log shows a list of all commits along with author name, date, commit message, and hash. It is useful for tracking changes and understanding the project’s history.

Example

git log

14. git diff — See the Difference Between Versions

git diff displays line-by-line changes between files or commits. It is used to review modifications before committing or to understand what changed between versions.

Example

git diff

15. git reset — Unstage or Undo Commits

git reset is used to undo commits or move files out of the staging area. It is helpful for correcting mistakes before sharing changes with others.

Example

git reset file.txt

16. git rm — Remove Files from Repo

git rm deletes files from the working directory and staging area. It is commonly used when cleaning up projects or removing obsolete files from version control.

Example

git rm oldfile.js

18. git config — Set Username & Email

git config stores your Git identity so commits show the correct author details in the history. Usually set once per machine using the --global option.

Example

git config --global user.name "Mitali"
git config --global user.email "mitali@gmail.com"
